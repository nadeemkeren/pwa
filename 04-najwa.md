**Rangkuman Rekayasa Perangkat Lunak (Poin Utama)**


**1. Teknologi & Pertukaran Data**

* Menggunakan **API** untuk menghubungkan antar sistem
* Format data: **JSON** dan **XML**
* Digunakan pada aplikasi web & mobile
* Memudahkan integrasi sistem secara cepat dan fleksibel


**2. Metodologi Pengembangan (SDLC)**

* **Waterfall:** linear (planning → analysis → design → implementation → testing → maintenance)
* **Agile:** iteratif dan fleksibel (sprint)
* **Scrum:** backlog, sprint, daily meeting
* **Extreme Programming (XP):** fokus kualitas kode & kolaborasi


**3. Pengujian Perangkat Lunak**

* **Black Box:** fokus input-output
* **White Box:** menguji kode program
* **Gray Box:** kombinasi keduanya
* **Penetration Testing:** uji keamanan sistem


**4. Estimasi Proyek**

* **SLOC:** berdasarkan jumlah baris kode
* **COCOMO:** estimasi biaya & waktu proyek
* Digunakan untuk perencanaan yang lebih terkontrol


**5. Peran dalam Tim**

* **Project Manager:** mengatur proyek
* **Developer:** membuat program
* **Designer:** desain UI/UX
* **QA/Tester:** memastikan kualitas
* **Marketing & Owner:** mendukung bisnis & tujuan proyek


**6. Siklus Pengembangan Sistem**

* Tahapan: planning → analysis → design → implementation → testing → maintenance
* Bersifat **iteratif (berulang)** terutama pada Agile
* Digunakan untuk peningkatan sistem secara terus-menerus


**7. Tools Pendukung**

* Web development tools
* Database
* Internet sebagai infrastruktur utama


**8. Implementasi pada Proyek LMS**

* **Planning:** menentukan modul (quiz, sertifikat, dll)
* **Analysis:** menentukan platform (web/desktop/mobile)
* **Tim:** owner, PM, developer, designer, dll
* **Teknologi:** HTML, JavaScript, API (JSON/XML), SQL/Cloudflare, GitHub
* **Design & Implementation:** desain UI dan coding sistem
* **Testing:** blackbox, whitebox, graybox, penetration testing
* **Dokumentasi SDLC:**

  * Requirement
  * Design
  * Technical
  * User
* **Maintenance:** pemeliharaan & pengembangan sistem
