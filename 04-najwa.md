Rangkuman Materi Rekayasa Perangkat Lunak

Materi Rekayasa Perangkat Lunak membahas berbagai aspek penting dalam pengembangan sistem, 

dimulai dari teknologi pertukaran data seperti API (Application Programming Interface) yang berfungsi sebagai penghubung antar aplikasi. 

API memungkinkan pertukaran data menggunakan format seperti JSON dan XML, sehingga integrasi antar sistem menjadi lebih cepat, efisien, dan fleksibel, terutama pada aplikasi web dan mobile.


Selanjutnya dibahas Software Development Life Cycle (SDLC) sebagai proses pengembangan perangkat lunak. 

Terdapat dua metode utama, yaitu Waterfall yang bersifat linear (planning, analysis, design, implementation, testing, maintenance) 

dan Agile yang bersifat iteratif melalui sprint, seperti pada framework Scrum yang menggunakan backlog, sprint, dan daily meeting. 

Selain itu, terdapat juga Extreme Programming (XP) yang menekankan kualitas kode dan kolaborasi tim.


Dalam pengembangan sistem, pengujian perangkat lunak menjadi tahap penting, yang meliputi Black Box Testing (berdasarkan input-output), 

White Box Testing (menguji kode program), dan Gray Box Testing (gabungan keduanya). Selain itu, terdapat juga pengujian keamanan seperti penetration testing.

Materi juga mencakup estimasi proyek menggunakan metode seperti SLOC (Source Lines of Code) dan COCOMO untuk memperkirakan waktu, biaya, dan sumber daya yang dibutuhkan 

agar proyek berjalan terencana. Selain itu, dijelaskan juga peran tim dalam proyek seperti Project Manager, Developer, Designer, QA, Tester, dan pihak lain yang saling bekerja sama.

Siklus pengembangan sistem bersifat iteratif, dimulai dari planning hingga maintenance, dan dapat terus berulang untuk meningkatkan kualitas sistem. 

Selain itu, penggunaan tools pendukung seperti web development tools, database, dan internet juga sangat penting dalam mendukung proses pengembangan.


Sebagai implementasi dari materi tersebut, pada proyek LMS, proses dimulai dari tahap planning dengan menentukan modul seperti quiz, sertifikat, dan desain sistem. 

Kemudian dilakukan analisis kebutuhan untuk menentukan platform (web, desktop, atau mobile). 

Setelah itu dibentuk tim yang terdiri dari owner, project manager, marketing, developer, dan desainer. 

Pada tahap perancangan, sistem dibangun menggunakan teknologi seperti HTML dan JavaScript dengan integrasi API berbasis JSON/XML serta database SQL atau layanan cloud seperti Cloudflare, dan hosting menggunakan GitHub. 

Selanjutnya masuk ke tahap desain dan implementasi, di mana developer melakukan coding dan desainer merancang antarmuka. 

Setelah sistem selesai, dilakukan pengujian menggunakan blackbox, whitebox, graybox, dan penetration testing untuk memastikan kualitas dan keamanan. 

Kemudian dibuat dokumentasi SDLC yang mencakup dokumentasi kebutuhan, desain, teknis, dan pengguna. 

Tahap terakhir adalah maintenance, yaitu pemeliharaan dan pengembangan sistem agar tetap optimal sesuai kebutuhan di masa depan.
