Rangkuman Materi Rekayasa Perangkat Lunak
1. Teknologi dan Pertukaran Data (API, JSON, XML)  
API digunakan sebagai penghubung antar aplikasi agar dapat bertukar data menggunakan format seperti JSON dan XML. Teknologi ini penting dalam pengembangan aplikasi web dan mobile karena memungkinkan integrasi sistem yang cepat dan efisien.

2. Metodologi Pengembangan Sistem (SDLC) 
SDLC memiliki dua pendekatan utama, yaitu Waterfall dan Agile. Waterfall bersifat linear dengan tahapan berurutan mulai dari planning hingga maintenance. Sedangkan Agile bersifat fleksibel dan iteratif melalui sprint, backlog, dan daily meeting, seperti pada Scrum dan Extreme Programming (XP).

3. Pengujian Perangkat Lunak (Software Testing)
Terdapat tiga jenis testing, yaitu Black Box (berdasarkan input-output), White Box (melihat struktur kode), dan Gray Box (kombinasi keduanya). Pengujian ini penting untuk memastikan sistem berjalan dengan baik.
4. Estimasi Proyek Software
Estimasi digunakan untuk menentukan waktu dan biaya proyek dengan metode seperti SLOC (berdasarkan jumlah baris kode) dan COCOMO (model perhitungan biaya dan durasi proyek).

5. Peran Tim dalam Proyek
Tim terdiri dari Project Manager, QA, Developer, Designer, dan Tester yang memiliki tugas masing-masing namun saling bekerja sama untuk mencapai tujuan proyek.

6. Siklus Pengembangan Sistem
Tahapan pengembangan meliputi planning, analysis, design, implementation, testing, dan maintenance. Proses ini dapat bersifat iteratif untuk meningkatkan kualitas sistem.

7. Tools dan Pendukung
Pengembangan software didukung oleh tools seperti web development, database, dan internet sebagai infrastruktur utama.
