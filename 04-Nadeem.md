
Rangkuman Materi Rekayasa Perangkat Lunak
1.Teknologi dan Pertukaran Data (API, JSON, XML)

Pada bagian awal materi dijelaskan mengenai teknologi yang digunakan dalam pengembangan sistem, khususnya API (Application Programming Interface) yang berfungsi sebagai penghubung antar aplikasi. API memungkinkan sistem untuk saling bertukar data menggunakan format seperti JSON dan XML. Penggunaan teknologi ini sangat penting dalam pengembangan aplikasi modern, terutama pada platform web dan mobile, karena memungkinkan integrasi antar sistem berjalan secara efisien, cepat, dan fleksibel.

2. Metodologi Pengembangan Sistem (Software Development Life Cycle / SDLC)
Materi selanjutnya membahas tentang siklus hidup pengembangan perangkat lunak atau SDLC. Terdapat dua pendekatan utama yang digunakan, yaitu metode Waterfall dan Agile. Metode Waterfall merupakan pendekatan tradisional yang bersifat linear dan berurutan, dimulai dari tahap planning (perencanaan), analysis (analisis kebutuhan), design (perancangan), implementation (pengembangan), hingga maintenance (pemeliharaan). Setiap tahap harus diselesaikan sebelum masuk ke tahap berikutnya.
Sebaliknya, metode Agile lebih fleksibel dan bersifat iteratif. Dalam Agile, pengembangan dilakukan secara bertahap melalui siklus kecil yang disebut sprint. Salah satu framework dalam Agile adalah Scrum, yang melibatkan backlog (daftar pekerjaan), sprint (periode kerja), serta daily meeting untuk memantau progres. Selain itu, terdapat juga Extreme Programming (XP) yang menekankan kualitas kode dan kolaborasi tim secara intensif.

3. Jenis-Jenis Pengujian Perangkat Lunak (Software Testing)
Dalam proses pengembangan software, pengujian merupakan tahap penting untuk memastikan sistem berjalan dengan baik. Terdapat tiga jenis pengujian utama yang dibahas, yaitu Black Box Testing, White Box Testing, dan Gray Box Testing. Black Box Testing dilakukan dengan menguji fungsi sistem berdasarkan input dan output tanpa melihat struktur internal kode. White Box Testing dilakukan dengan menganalisis dan menguji bagian dalam kode program secara langsung. Sedangkan Gray Box Testing merupakan kombinasi dari keduanya, di mana penguji memiliki sebagian pengetahuan tentang sistem untuk menghasilkan pengujian yang lebih efektif.

4. Estimasi Proyek Perangkat Lunak
Materi juga mencakup cara melakukan estimasi dalam proyek pengembangan software. Estimasi ini penting untuk menentukan waktu, biaya, dan sumber daya yang dibutuhkan. Metode yang digunakan antara lain SLOC (Source Lines of Code), yaitu menghitung jumlah baris kode sebagai dasar estimasi, serta COCOMO (Constructive Cost Model) yang merupakan model matematis untuk memperkirakan biaya dan durasi proyek berdasarkan ukuran dan kompleksitas sistem. Dengan estimasi yang tepat, proyek dapat berjalan lebih terencana dan terkontrol.

5. Peran dan Struktur Tim dalam Proyek Software
Dalam pengembangan perangkat lunak, terdapat beberapa peran penting yang saling bekerja sama. Project Manager (PM) bertugas mengatur jalannya proyek dan memastikan semua berjalan sesuai rencana. Quality Assurance (QA) bertanggung jawab terhadap kualitas sistem. Developer bertugas mengembangkan dan menulis kode program. Designer merancang tampilan dan pengalaman pengguna (UI/UX). Tester bertugas melakukan pengujian untuk menemukan kesalahan sebelum sistem digunakan. Setiap peran memiliki tanggung jawab yang berbeda namun saling melengkapi dalam mencapai tujuan proyek.

6. Siklus dan Proses Pengembangan Sistem
Materi juga menggambarkan siklus pengembangan perangkat lunak dalam bentuk diagram melingkar yang menunjukkan bahwa proses ini bersifat berulang (iteratif). Tahapan yang dilalui meliputi planning, analysis, design, implementation, testing, dan maintenance. Dalam pendekatan modern seperti Agile, siklus ini tidak berhenti pada satu putaran saja, melainkan terus diulang untuk meningkatkan kualitas sistem dan menyesuaikan dengan kebutuhan pengguna yang terus berkembang.

7. Tools dan Pendukung Pengembangan   
Selain konsep utama, materi juga menyinggung penggunaan berbagai tools dan teknologi pendukung seperti web development tools, database, serta internet sebagai infrastruktur utama. Hal ini menunjukkan bahwa pengembangan software tidak hanya bergantung pada metode, tetapi juga pada teknologi yang digunakan untuk mendukung proses tersebut.
