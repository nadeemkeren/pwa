Rangkuman Materi (Manajemen Proyek & Pengembangan Sistem)
1. Konsep Dasar Pengembangan Sistem
Ada pembahasan tentang SDLC (Software Development Life Cycle)
Metode yang disebut:
Waterfall (tahapan berurutan)
Scrum / Agile (iteratif & fleksibel)
2. Metode Pengembangan
Waterfall
Tahapan jelas dan berurutan
Scrum (Agile)
Sprint
Iterasi cepat
Lebih fleksibel terhadap perubahan
3. Peran dalam Proyek
Tertulis beberapa role:
PM (Project Manager) → mengatur proyek
Tim → mengerjakan tugas teknis
Owner / Client → pemberi kebutuhan
4. Siklus Waktu / Aktivitas
Ada pembagian waktu kerja:
Time
Cost
Quality
Delivery
(ini biasanya disebut project constraints / batasan proyek)
5. Proses & Tahapan
Beberapa tahapan yang terlihat:
Planning
Analysis
Design
Implementation
Maintenance
6. Tools & Konsep Tambahan
Blackbox & Whitebox testing
Coding
UI/UX
Database
Performance
Debugging
7. Diagram Lingkaran (Siklus)
Menunjukkan proses berulang:
Plan → Do → Check → Improve
(semacam siklus perbaikan berkelanjutan)
8. Kesimpulan
Materi ini membahas: 👉 Cara mengembangkan software dari awal sampai selesai
👉 Metode kerja tim (Waterfall vs Agile/Scrum)
👉 Peran dalam proyek
👉 Pentingnya testing, coding, dan evaluasi
