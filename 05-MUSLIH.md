
1. SDLC (Software Development Life Cycle)
Papan tulis mencantumkan tahapan standar dalam pengembangan perangkat lunak:

Plan: Perencanaan awal.

Analysis: Analisis kebutuhan (disebutkan Dict, BP).

Design: Perancangan sistem.

Implementation: Pengkodean (menggunakan PWA, JS, HTML).

Testing: Pengujian perangkat lunak.

Docs: Pendokumentasian.

Maintenance: Pemeliharaan sistem.

2. Metodologi Pengembangan
Ada pembagian antara metode tradisional dan modern:

Waterfall: Linear dan kaku.

Prototype: Berbasis pembuatan prototipe.

Agile: Fokus pada fleksibilitas, mencakup Scrum dan XP (Extreme Programming).

3. Pembagian Tugas Kelompok (K1, K2, K3)
Terdapat pembagian proyek berdasarkan teknologi frontend:

K1: Proyek POS (Point of Sale) menggunakan Vue.

K2: Proyek LMS (Learning Management System) menggunakan React.

K3: Proyek LMS menggunakan Svelte.

4. Peran dalam Proyek (Stakeholders)
Disebutkan hierarki atau daftar peran:

Owner / Client

PM (Project Manager)

User / Member

Admin

Dev (Developer)

5. Catatan Teknis Lainnya
Repository: Link GitHub (github.com/sismadi/PWA).

PWA: Progressive Web Apps menjadi fokus utama implementasi.

POS Details: Disebutkan elemen seperti Payment, Product, QR, dan QC.

Pertanyaan Terkait Materi
Untuk memastikan kamu memahami diskusi di papan tulis tersebut, coba jawab pertanyaan-pertanyaan berikut:

Metodologi: Apa perbedaan mendasar antara penggunaan metode Waterfall dan Agile (Scrum) dalam konteks pengembangan PWA yang ada di catatan tersebut?

Teknologi: Jika kelompok 1 menggunakan Vue dan kelompok 2 menggunakan React, apa tantangan terbesar yang mungkin dihadapi saat mengimplementasikan fitur PWA pada masing-masing framework tersebut?

Peran: Mengapa peran Owner/Client diletakkan di nomor urut pertama dalam daftar peran proyek? Apa dampaknya jika Owner tidak terlibat dalam tahap Analysis?

SDLC: Pada tahap Testing, terdapat catatan kecil mengenai "SQL". Menurutmu, apa yang sedang diuji dalam kaitannya dengan database pada tahap tersebut?

Workflow: Apa maksud dari instruksi -b Fork garpu yang tertulis di dekat link GitHub tersebut dalam alur kerja kolaborasi tim?
