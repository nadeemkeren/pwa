1. Teknologi & Konversi Data
Terdapat pembahasan tentang:
API → JSON → XML
Artinya:
Sistem dapat saling terhubung menggunakan API
Data dikirim dalam format seperti JSON atau XML
Ini biasanya digunakan dalam pengembangan sistem berbasis web / aplikasi modern
2. Sistem / Platform yang Digunakan
Disebutkan beberapa platform:
PHP
Selenium
CMS
Fungsi:
PHP → backend development
Selenium → testing otomatis
CMS → manajemen konten
3. Metodologi Pengembangan (SDLC)
Ada bagian SDLC (Software Development Life Cycle) dengan tahapan:
Planning
Scenarios
Maintenance
(kemungkinan juga: testing & development)

➡️ Intinya:
SDLC adalah siklus pengembangan software dari awal sampai maintenance

4. Model Pengembangan
Beberapa model disebutkan:
Waterfall
Scrum
XP (Extreme Programming)
Penjelasan:
Waterfall → linear (berurutan)
Scrum → agile berbasis sprint
XP → fokus pada coding cepat & kolaborasi
5. Estimasi Proyek (COCOMO & SLOC)
Ada pembahasan:
SLOC (Source Lines of Code)
COCOMO
Digunakan untuk:
Menghitung estimasi:
Waktu
Biaya
Usaha (effort)
6. Faktor Estimasi
Faktor yang mempengaruhi:
Time (waktu)
Effort (usaha)
Size (ukuran program)
Cost (biaya)
7. Analisis & Desain Sistem
Tahapan:
Basic
Intermediate
Detail
Artinya:
Sistem dianalisis dari umum ke detail sebelum dibuat
8. Agile & Implementasi
Disebutkan:
Agile
Contoh: Umbrella University
Agile:
Metode fleksibel
Cepat beradaptasi dengan perubahan
9. Konsep Diagram / Loop
Ada gambar lingkaran (flow):
Menunjukkan proses iterasi (berulang)
Biasanya terkait:
Agile / Scrum cycle
Kesimpulan Utama

Materi ini membahas alur lengkap pengembangan software:

Teknologi & format data (API, JSON, XML)
Tools & platform (PHP, Selenium, CMS)
Proses SDLC
Model pengembangan (Waterfall, Scrum, XP)
Estimasi proyek (SLOC, COCOMO)
Analisis & desain sistem
Implementasi dengan metode Agile
